# 00. プロジェクト概要 — APPALE!

## ビジョン

日本企業・日本人が、**日本語だけ**で世界へ情報発信・販売できるAIプラットフォーム。
語学力や海外展開のノウハウがないことをボトルネックにせず、良い商品・情報を持つ人が世界中の顧客に届けられる状態を作る。

## ターゲットユーザー

- 海外展開したいが英語・多言語対応のリソースがない中小企業・個人事業主
- 越境ECを始めたいがマーケティング／ローカライズが障壁になっているセラー
- 情報発信（ブログ・SNS・製品情報など）を多言語で行いたいがコストが見合わないクリエイター・企業

## 課題

- 翻訳だけでは不十分（文化的文脈・現地の商習慣・法規制への配慮が必要）
- 多言語での継続的な情報発信・更新の運用コストが高い
- 海外向けの販売チャネル（EC・マーケットプレイス・SNS）ごとに個別対応が必要

## ソリューション概要

ユーザーは日本語で情報を入力・更新するだけで、AIが以下を担う：

1. 多言語への翻訳・ローカライズ（文化的文脈を考慮）
2. 各種チャネル（Web・SNS・ECモール等）向けフォーマットへの変換
3. 発信・出品の実行、および継続的な更新の同期

## コア機能（想定・要検証）

- [ ] 日本語コンテンツの入力・管理（テキスト／商品情報／画像など）
- [ ] 多言語ローカライズエンジン（翻訳＋文化的適応）
- [ ] チャネル連携（EC・SNS・自社サイト等への配信）
- [ ] 販売機能（決済・在庫・注文管理との連携）
- [ ] ダッシュボード（発信状況・反応・売上の可視化）

> 上記は現時点での仮説であり、優先順位・スコープは今後のヒアリング・検証で変わりうる。

## 技術スタック（暫定）

- フロントエンド: TypeScript（Next.js を想定）
- バックエンド: Python（AI／ローカライズ処理と親和性を重視）
- 詳細な構成・DB・インフラは未確定。決定した内容は本ドキュメントおよび `CLAUDE.md` を更新すること。

## 非スコープ（当面やらないこと）

- 独自の決済基盤の開発（既存決済サービスとの連携を優先）
- 完全自動・無人での海外法規制対応の保証（人間によるレビューを前提とする）

## オープンな論点

- MVPで対応する最初のチャネル（EC / SNS / 自社サイトのどれから始めるか）
- 対応言語の優先順位
- マネタイズモデル（サブスク／従量課金／成果報酬）
- 個人情報・越境データ移転に関する法的要件の整理
