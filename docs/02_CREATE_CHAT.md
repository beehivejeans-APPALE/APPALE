# 02. 企画ヒアリングチャット（Phase 2）

Phase 2 では、企画を日本語対話でヒアリングするチャット画面を実装する。今回のスコープは「自然な対話ができる状態まで」であり、会話からの構造化データ抽出・クラファンページ生成は対象外。

## 動作確認

2026-07-02、ローカル環境（`npm run dev`）で以下を確認した。

- 未ログイン状態で `/create` にアクセスすると `/login` へリダイレクトされる（Server Component での判定、`307`）。
- 未ログイン状態で `/api/chat` に直接 POST すると `401 Unauthorized` が返る。
- （認証チェックを一時的に無効化した状態で）実際に Gemini API を呼び出し、ユーザーの回答内容を踏まえた自然な追加質問が返ってくることを確認した。複数ターンにわたり会話の文脈（直前の回答）を踏まえた応答になっている。
- チャットUIは、ユーザーの発言が右寄せ（黒背景）、AIの発言が左寄せ（グレー背景）の吹き出し形式で表示される。

マジックリンク経由の実ブラウザでのログイン → `/create` での一連の操作は、認証フローの性質上（メール受信が必要）今回は自動検証できていない。次回、実際にログインした状態で目視確認することを推奨する。

## 画面構成

| パス | 種別 | 内容 |
| --- | --- | --- |
| `/create` | Server Component（`src/app/create/page.tsx`） | `supabase.auth.getUser()` でログイン判定し、未ログインなら `/login` へリダイレクト。ログイン済みなら `ChatPanel` を描画する。 |
| `src/app/create/ChatPanel.tsx` | Client Component | チャットUI本体。メッセージ一覧・入力欄・送信ボタンを持つ。 |
| `src/app/api/chat/route.ts` | Route Handler（POST） | ログイン判定後、会話履歴を Gemini API に渡し、AIの返答を返す。 |

### チャットUIの挙動

- 画面表示時、AIからの最初の質問（企画の目的を尋ねる固定メッセージ）を表示する。Gemini APIは呼ばない。
- ユーザーがメッセージを送信すると、それまでの会話履歴全体（`ChatMessage[]`）を `/api/chat` に POST する。
- 応答が返るまでは「考え中...」を表示し、入力・送信を無効化する。
- 通信エラー時は吹き出しの下にエラーメッセージを表示する。
- 会話はブラウザの状態（React state）にのみ保持し、DB等への永続化は今回は行わない。

## AIへのプロンプト設計

`src/app/api/chat/route.ts` 内の `SYSTEM_INSTRUCTION` として実装。ヒアリング対象は以下の4項目：

1. 企画の目的（何を実現したいか）
2. ターゲット（誰に向けたプロジェクトか）
3. ストーリー（背景・想い）
4. リターン設計（支援者に何を返すか）

設計方針：

- 一度に1つだけ質問し、フォームの穴埋めのような機械的な質問にしない。
- 直前の回答を踏まえて自然に次の質問へつなげる（4項目の順序には固執しない）。
- 回答が曖昧・簡潔すぎる場合は深掘りしてよい。
- 4項目をひととおり聞けたら要約し、感謝を伝えた上で「ページ生成は準備中」であることを伝える（実際のページ生成はまだ実装しないため）。
- 常に日本語、丁寧だが堅すぎない温かみのある口調。
- 会話に専念し、企画書やページ文章そのものは生成しない（Phase 2のスコープ外）。

## 技術構成

- Gemini API 呼び出しに `@google/genai`（Google Gen AI SDK for TypeScript/JavaScript）を使用。
  - `GEMINI_API_KEY`（`.env.local`、サーバー専用・`NEXT_PUBLIC_` プレフィックスなし）を使用してクライアントを生成。
  - モデルは `gemini-2.5-flash` を使用（会話用途のため軽量・高速なモデルを選定）。
  - `ai.models.generateContent()` に会話履歴全体（`role: "user" | "model"` の配列）と `config.systemInstruction` を渡し、都度リクエストとして呼び出すステートレスな方式（サーバー側でセッションを保持しない）。
- `/api/chat` は Route Handler として実装し、`src/lib/supabase/server.ts` の `createClient()` で `supabase.auth.getUser()` を呼び、未ログインなら `401` を返す（[`docs/01_AUTH.md`](01_AUTH.md) のログイン判定方針に準拠）。
- `/create` の認証チェックは Server Component（`page.tsx`）内で行い、未ログインなら `redirect("/login")`。
- 会話メッセージの型 `ChatMessage`（`role: "user" | "model"`, `content: string`）は `src/types/chat.ts` に定義し、`ChatPanel.tsx` と `route.ts` の両方で共有する。

## 今後の拡張予定（オープンな論点）

- 会話からの構造化データ抽出（4項目をJSON等に整理する処理）は未実装。次フェーズで着手する。
- クラファンページ生成（構造化データを元にしたページ文章・構成案の生成）は未実装。
- 会話履歴の永続化（DB保存、途中離脱からの再開）は未実装。今回はブラウザのState内のみ。
- Gemini APIのレスポンス遅延・エラー時のリトライ方針は未検討。
